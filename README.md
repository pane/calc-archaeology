# calc-archaeology

> Historical calculator firmware, algorithms, hardware behavior, and reverse engineering research

## Name

The project uses the spelling "archaeology" in the repository name.
The alternative spelling "archeology" may also appear in references,
historical notes, and search terms.

## Purpose

`calc-archaeology` is a research-oriented archive focused on understanding
how classic calculators actually worked internally.

The project combines:
- historical preservation
- firmware reverse engineering
- ROM analysis
- emulator development
- execution tracing
- algorithm reconstruction
- hardware observation
- AI-assisted comparative analysis

The repository is intended to support both human researchers and future
AI-assisted analysis systems.

## Core Rule

Every technical statement in this repository must be traceable to either:

- primary evidence, or
- an explicitly labeled hypothesis.

Placeholder citations are not allowed. If evidence has not yet been cataloged,
the statement must remain tentative or be omitted.

## Current Areas of Interest

- Texas Instruments calculator families
- TMS0500 series systems
- HP classic calculators
- OKI-based systems
- Numerical algorithms (sqrt, ln, trig, AOS, etc.)
- Firmware timing behavior
- ROM disassembly and symbolic reconstruction
- Emulator validation
- Cross-platform algorithm comparisons

## Research Philosophy

This repository distinguishes explicitly between:
- primary evidence
- observations
- interpretations
- hypotheses
- AI-assisted reasoning

Primary artifacts and experimentally observed behavior always take
precedence over interpretation.

See:
- `docs/fundamental-principles.md`
- `docs/research-methodology.md`
- `docs/rom-annotation-conventions.md`
