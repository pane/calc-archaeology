# Intake

This directory describes how unsorted source material enters the project.

Raw copyrighted, private, or license-unclear material should normally remain
outside the public repository. This directory is for intake notes, indexes, and
derived summaries that are safe to publish.

## Workflow

1. Record the source in an intake note or evidence candidate list.
2. Identify origin, author, date, and publication status when possible.
3. Classify the material by source type.
4. Extract only conservative summaries unless publication rights are clear.
5. Promote the item to an evidence record only when the record can be cited
   accurately.

## Source Types

- ROM
- emulator trace
- hardware observation
- manual or manufacturer documentation
- patent
- published research
- prior chat
- AI-generated draft
- unknown

## Rule

Messy material may be stored and sorted, but uncertain conclusions must not be
published as facts.
